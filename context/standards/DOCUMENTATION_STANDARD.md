# Documentation Standard

## 1. Purpose
Ensure system context is machine-readable and human-navigable.

## 2. Scope
Applies to all files in the `context/` directory.

## 3. Rules & Guidelines
- All documentation must be written in Markdown.
- Every file must contain an executable YAML frontmatter block.
- Keep content concise. Avoid unbounded essays.
- Documentation must reside within the `context/` foundation directory.
