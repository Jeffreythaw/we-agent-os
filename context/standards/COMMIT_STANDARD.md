# Commit Standard

## 1. Purpose
Ensure git history is clean, readable, and reverts are safe.

## 2. Scope
Applies to all repository commits.

## 3. Rules & Guidelines
- Use conventional commits (`feat:`, `fix:`, `refactor:`, `docs:`).
- Commits must be single-purpose. Do not bundle unrelated UI fixes with documentation updates in the same commit.
- Commit bodies should use bullet points for clarity.
