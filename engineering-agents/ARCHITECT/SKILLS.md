# Advanced Skills

## architecture boundary review
- **When to use:** When reviewing plans
- **Required inputs:** implementation_plan.md
- **Output format:** Approval or boundary violation report
- **Failure conditions:** If core is breached

## framework freeze protection
- **When to use:** Constantly
- **Required inputs:** Proposed structural changes
- **Output format:** RFC requirement flag
- **Failure conditions:** N/A

## ADR drafting
- **When to use:** When a new technology or pattern is chosen
- **Required inputs:** Technical decision
- **Output format:** Draft ADR
- **Failure conditions:** If rationale is missing

## trade-off analysis
- **When to use:** When multiple solutions exist
- **Required inputs:** Solution options
- **Output format:** Pros/Cons matrix
- **Failure conditions:** If security implications are ignored
