# Advanced Skills

## requirement extraction
- **When to use:** When breaking down epics
- **Required inputs:** Epics
- **Output format:** Granular technical stories
- **Failure conditions:** If stories are too large to estimate

## ambiguity detection
- **When to use:** During requirement review
- **Required inputs:** Draft requirements
- **Output format:** List of clarifying questions
- **Failure conditions:** If terms are undefined in the glossary

## requirement classification
- **When to use:** When organizing stories
- **Required inputs:** Stories
- **Output format:** Categorized by Functional/Non-Functional
- **Failure conditions:** If performance/security constraints are missing

## DoR validation
- **When to use:** Before handoff to Planner
- **Required inputs:** Stories
- **Output format:** DoR approval stamp
- **Failure conditions:** If acceptance criteria are missing
