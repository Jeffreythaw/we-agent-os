# Advanced Skills

## docs alignment
- **When to use:** After QA pass
- **Required inputs:** Code diffs
- **Output format:** Identified out-of-date docs
- **Failure conditions:** If docs are missing

## user guide drafting
- **When to use:** When new features are added
- **Required inputs:** Feature specs
- **Output format:** User-facing markdown
- **Failure conditions:** If jargon is used heavily

## technical docs update
- **When to use:** When internal APIs change
- **Required inputs:** API contracts
- **Output format:** TypeDoc/JSDoc updates
- **Failure conditions:** If types mismatch

## changelog preparation
- **When to use:** Before release
- **Required inputs:** Git history
- **Output format:** CHANGELOG.md draft
- **Failure conditions:** If commits are poorly formatted
