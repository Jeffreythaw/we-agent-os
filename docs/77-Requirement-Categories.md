# Requirement Categories

The Requirement Analyzer maps all raw project facts into discrete categories to accurately compute the Requirement Score.

## Core Categories
- **Functional**: What the software must do (e.g., "User can log in", "System exports PDF").
- **Non-Functional**: Performance and scalability constraints (e.g., "Load time < 2s").
- **Technical**: The tech stack and architectural constraints (e.g., "Node.js", "PostgreSQL").
- **Design**: UI/UX constraints, wireframes, and styling rules (e.g., "Tailwind CSS", "Dark Mode").
- **Operations**: Deployment and CI/CD rules (e.g., "Dockerize", "Deploy to Vercel").
