# Project Artifact Map

During the Business Analysis Phase and subsequent Bootstrapping, the OS generates deterministic markdown artifacts outlining the project agreement.

## BA Artifacts

1. **`Q_AND_A.md`**
   - **Purpose**: A historical transcript of the questions asked by the Question Engine and the answers provided by the user. Ensures traceability of design decisions.
2. **`REQUIREMENTS.md`**
   - **Purpose**: A human-readable summary of all facts categorized by the Requirement Analyzer. Acts as the business sign-off document.
3. **`PROJECT_SPEC.md`**
   - **Purpose**: The technical architecture document generated only *after* DoR is met. It dictates the boundaries for the Execution phase.
