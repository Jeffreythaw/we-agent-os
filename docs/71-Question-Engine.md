# Question Engine

## Purpose
The Question Engine bridges the gap between incomplete requirements and the strict Definition of Ready. 

## Responsibilities
1. **Identify Gaps**: Reads the `warningCount` and missing requirement violations generated by the Requirement Score evaluation.
2. **Formulate Queries**: Deterministically prompts the user for missing variables.
3. **Iterative Resolution**: Suspends the BA workflow, awaits user input, and feeds the new facts back into the Requirement Analyzer.

## AI-Optional Design
The Question Engine does not require an LLM. It maps deterministic missing facts (e.g., `missing_database_schema`) to predefined human-readable questions (e.g., "What database technology should be used for this project?"). AI is only utilized to make the tone conversational if the AI Escalation Policy permits it.
